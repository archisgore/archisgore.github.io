Now consider the average question I get at networking events, on [Quora](https://www.quora.com), and from friends and family:

> *“I want to create my own website or blog, but I want to host it myself.”*

Hooboy…. where do we begin? VMs, Load Balancers, Cloud Providers, Stable IP addresses, Domain Names, CMS systems (or not), etc.

This is a LARGE concept loop because each of these concepts begets more questions, more “what ifs”, more “what about”, more “but whys” and just more more more.

So let me break down what a Closed Concept Loop is.

#### Concept Loop

A concept loop is an entire loop you have to follow (a technological and cognitive obstacle course) when you have to get the complete project done.

Why a loop? Because I’ve done this a thousand times, but the next time someone needs to host a website under those stated constraints, the loop MUST BE FOLLOWED. There are no shortcuts. We loop and loop and loop.

#### Closed Loop

The Closed comes from “closures” or “closed context”. Meaning, if the loop is infinite, nothing ever works. This is undesirable for everyone. The cloud provider, CPU maker, memory maker, rack maker, hypervisor maker, switch maker, load balancer maker, etc. etc. etc. makes a few cents
